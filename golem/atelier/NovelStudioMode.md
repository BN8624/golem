# Golem Atelier Mode 제안서 (소설 스튜디오)

## 한 줄

골렘 Studio(가상 게임회사)의 역할순환·계약패킷·다축리뷰 구조를 **소설 작가 공방**으로 옮긴다.
게임의 "굴러가나 → 정답이 맞나(exact-match oracle)"를 소설의 **"캐논이 맞나"** 로 재현한다.

## 골렘에서 그대로 가져오는 것 (옮겨짐)

1. 역할 순환 워커풀 — 11슬롯이 단계마다 기획→설계→specQA→집필→통합으로 책상을 옮긴다.
2. lead 1 + reviewer 10 (다축 강제) — 리뷰어 10명이 서로 다른 단일 축만 사냥한다.
3. 계약 패킷 인수인계 — 단계 간 소통은 대화가 아니라 FROZEN 문서(바이블·아웃라인)로만.

## 옮겨지지 않는 한 가지 — oracle (이 설계의 전부)

골렘의 힘은 exact-match 골든이다. 소설엔 "이 문장이 정답"의 골든이 없다. 그래서 oracle을 두 층으로 쪼갠다.

| 층 | 채점 | 처리 |
|---|---|---|
| **캐논** (연속성·세계규칙·인물사실·복선회수·타임라인) | exact 가능 | 골렘 oracle 그대로. `canon_check.py` |
| **미학** (문장·긴장·페이싱·목소리) | exact 불가 | 합의채점 흉내 금지. 다축 진단 + 사용자=최종 oracle |

> **경계는 미정**이다 — `canon_check.py`를 실데이터로 돌려보고 "뭐가 기계로 잡히나"로 선을 긋는다.

## 단계 매핑

| 골렘 studio | atelier | reviewer 10축 예시 |
|---|---|---|
| planning | 로그라인 → 스토리 바이블 | 동기 없는 행동 / 회수 안 되는 테마 / 타임라인 모순 / 톤 불일치 |
| design | 비트시트·챕터 아웃라인 | 복선 미회수(setup→payoff) / 독자가 모르는 정보에 의존하는 반전 / 분량 불균형 |
| specQA | 씬 계약 + 캐논/미학 위험 분리 | 검증 불가한 기준 = 미학으로 격리 |
| build | 같은 씬 병렬 초고 | — |
| integration | 캐논 게이트 통과 초고 선정 + 미학은 사용자 | — |
| adversarial | "회의적 독자"가 구멍·지루함 공격 | — |

## 작은 아이디어 키우기

골렘 `bank_add_*_p0..p4`(단계적 기능 추가)의 소설판 — 같은 파이프라인을 층을 쌓듯 반복한다.
p0 전제 → p1 세계관 → p2 인물 → p3 서브플롯. 매 패스 산출이 **캐논 원장**에 누적되고,
다음 패스는 그 FROZEN 캐논을 계약으로 받는다.

## 현재 위치

`canon_check.py` 1개만 포팅 완료(캐논 채점기). 나머지 단계는 그 위에 얹는다.
