# Forge 작업 지침

- Forge는 `C:\Users\USER\golem\golem\forge` 안에서만 실행되는 독립 프로젝트다.
- `atelier`와 다른 형제 프로젝트를 import하거나 런타임 경로로 참조하지 않는다.
- Atelier 자료는 `reference/legacy`에 복사한 사본만 참고하며 수정하지 않는다.
- 목표 장편은 5권이다.
- 구조화 문서가 유일한 정본이며 산문은 마지막 단계에서 생성하는 파생 산출물이다.
- 계층은 시리즈, 권, 큰 사건, 장면 순서로 고정한다.
- 설정 변경, 복선, 회수는 각각 정확히 하나의 소유 문서가 있어야 한다.
- 독립 검증을 통과한 후보만 정본으로 승격하고, 그 뒤에만 상태 원장을 갱신한다.
- 산문 오류를 직접 고치지 않고 원인이 된 계약, 프롬프트, 검증기를 강화한 뒤 해당 범위를 재생성한다.
- 산문 생성은 스키마, 추적성, 상태 연속성 검증이 준비된 뒤 시작한다.
- 새 소스 파일 첫 줄에는 역할을 설명하는 한 줄짜리 한국어 주석을 둔다.
- 비자명한 작업 전에는 `checklist.md`와 `context-notes.md`를 갱신한다.
- 코드 변경 후 관련 테스트 또는 최소한 구문 검사를 실행하고 결과를 기록한다.
