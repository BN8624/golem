# Forge 핸드오프

## 현재 상태

Forge 독립 프로젝트 생성과 1차 구조 검증기 구현이 끝났다. 아직 모델 호출이나 신규 원고 생성은 시작하지 않았다.

## 완료 항목

- 5권 구조를 강제하는 시리즈, 권, 사건, 장면, 상태 원장 스키마.
- 계층, 순번, 상태 연속성, 이전 장면 연결 검증.
- 설정 변경, 복선, 회수의 단일 소유권 검증.
- 복선보다 회수가 먼저 등장하는 구조 차단.
- 숨은 모델 폴백 제거.
- Atelier 핵심 자료의 읽기 전용 참고 사본 보관.

## 다음 구현 순서

1. 구조 문서 후보를 임시 영역에 생성한다.
2. 독립 검증을 통과한 후보만 `story` 정본으로 원자적 승격한다.
3. 승격 결과에서 상태 원장을 재구성하고 재실행 결과가 같은지 검사한다.
4. 5권 전체의 신규 시리즈 초안을 구조 문서로 생성한다.
5. 권, 사건, 장면 순서로 세분화한 뒤 산문 생성을 시작한다.

## 실행 명령

```powershell
python -m unittest discover -s tests -v
python pipeline\validate_structure.py
```

현재 `story/series.json`이 없으므로 두 번째 명령은 신규 구조 문서를 만든 뒤 사용하는 명령이다.
